# PTM功能块说明文档

## 1. 功能块概述

PTM（Pulse Time Measurement）功能块是一个用于测量输入信号高电平持续时间的功能块。它使用TON（通电延时定时器）来实现计时功能，能够准确测量输入信号从变为高电平到变为低电平之间的时间间隔。

## 2. 功能块定义

```st
FUNCTION_BLOCK PTM
  VAR_INPUT
    IN : BOOL;
  END_VAR
  VAR_OUTPUT
    ET : TIME;
  END_VAR
  VAR
    TM : TIME;
    TON1 : TON;
    M : BOOL := FALSE;
  END_VAR

  TON1(
       IN := IN,
       PT := T#10S);
       
  IF IN = TRUE THEN
    TM := TON1.ET;
  ELSIF M = TRUE THEN
    ET := TM;
  ELSE
    M := TRUE;
  END_IF;
END_FUNCTION_BLOCK
```

## 3. 参数说明

### 3.1 输入参数

| 参数名称 | 类型 | 描述 |
|---------|------|------|
| IN | BOOL | 输入信号，用于触发和停止计时 |

### 3.2 输出参数

| 参数名称 | 类型 | 描述 |
|---------|------|------|
| ET | TIME | 输出信号高电平持续时间 |

### 3.3 内部变量

| 变量名称 | 类型 | 描述 |
|---------|------|------|
| TM | TIME | 临时存储定时器的已用时间 |
| TON1 | TON | 通电延时定时器，用于计时 |
| M | BOOL | 状态标志，用于指示是否可以开始测量。当检测到一个低电平后，M 变为 TRUE，此时高电平的计时测量才有效 |

## 4. 工作原理

PTM功能块的工作原理基于TON通电延时定时器，具体如下：

1. **定时器配置**：
   - TON1定时器的输入IN连接到外部输入信号IN
   - 定时器的预设时间PT设置为T#10S，作为最大测量时间限制

2. **高电平计时**：
   - 当输入信号IN为TRUE时，TON1开始计时
   - 此时将TON1的已用时间（ET）实时存储到变量TM中
   - 随着时间推移，TM会持续更新为最新的已用时间值

3. **高电平结束处理**：
   - 当输入信号IN变为FALSE时：
     - 首先检查状态标志M是否为TRUE
     - 如果M为TRUE，则将TM的值赋给输出ET，此时ET就是高电平的持续时间
     - 如果M为FALSE（第一次运行时），则将M设置为TRUE，不输出持续时间

4. **状态标志管理**：
   - 状态标志M用于指示是否可以开始测量
   - 初始值为FALSE，表示尚未检测到低电平，此时测量结果不可用
   - 第一次输入信号变为FALSE时设置为TRUE，表示已经检测到低电平，此时高电平的计时测量才有效
   - 之后每次输入信号变为FALSE时，都会将TM的值赋给输出ET

## 5. 工作流程

1. **首次运行**：
   - M = FALSE，表示尚未检测到低电平，测量结果不可用
   - 当IN变为TRUE时，TON1开始计时，TM记录ET值
   - 当IN变为FALSE时，M设置为TRUE，表示已经检测到低电平，此时高电平的计时测量才有效，不输出ET值

2. **后续运行**：
   - M = TRUE，表示已经检测到低电平，测量结果有效
   - 当IN变为TRUE时，TON1重新开始计时，TM重新记录ET值
   - 当IN变为FALSE时，TM的值（即高电平持续时间）被赋给ET输出

## 7. 技术要点

1. **定时器特性**：
   - TON（通电延时定时器）在输入为TRUE时开始计时，输入为FALSE时停止计时并复位
   - 其ET（已用时间）属性实时反映当前计时值

2. **状态管理**：
   - 使用状态标志M来区分首次运行和后续运行
   - 确保从第二次测量开始才能获得有效输出

3. **测量精度**：
   - 测量精度取决于PLC的扫描周期
   - 定时器的计时精度通常与PLC的系统时钟精度一致

4. **测量范围**：
   - 最大测量时间由TON1的PT值决定，当前设置为10秒
   - 最小可测量时间取决于PLC的扫描周期

## 8. 应用场景

PTM功能块适用于需要测量输入信号高电平持续时间的场景，例如：

- **按钮按下时间测量**：测量用户按下按钮的持续时间
- **传感器信号脉冲宽度测量**：测量传感器输出脉冲的宽度
- **设备运行时间监控**：监控设备启动信号的持续时间
- **事件持续时间记录**：记录特定事件的持续时间
- **工艺参数控制**：根据信号持续时间控制工艺参数

## 9. 注意事项

1. **首次测量**：
   - 第一次输入信号变为FALSE时不会输出持续时间，因为M初始值为FALSE
   - 从第二次测量开始，每次输入信号变为FALSE时都会输出持续时间

2. **状态标志M的作用**：
   - M用于指示是否可以开始测量
   - 初始值为FALSE，表示尚未检测到低电平，此时测量结果不可用
   - 只有检测到一个低电平后，M变为TRUE，此时高电平的计时测量才有效
   - 如果没有一个低电平，会无法判断高电平的开始时间，测量的结果不可用

3. **测量范围**：
   - 最大可测量时间为10秒，超过此时间会被截断
   - 如果需要测量更长时间，应增加TON1的PT值

4. **复位行为**：
   - 当输入信号IN变为FALSE后，ET输出会保持最后一次测量的值
   - 直到下一次输入信号从TRUE变为FALSE时，ET才会更新

5. **精度考虑**：
   - 测量精度受PLC扫描周期影响，扫描周期越长，测量精度越低
   - 对于高精度测量需求，应选择扫描周期较短的PLC系统

## 10. 改进建议

1. **初始化M为TRUE**：
   - 这样第一次输入信号变为FALSE时就能输出持续时间

2. **增加RUNNING输出**：
   - 添加RUNNING输出，指示是否正在测量高电平持续时间

3. **增加最大测量时间参数**：
   - 将PT值改为可配置参数，以适应不同的测量需求

4. **优化逻辑**：
   - 使用上升沿和下降沿触发器来更准确地捕获信号变化

5. **添加复位功能**：
   - 添加复位输入，允许手动复位测量结果

6. **增加超时检测**：
   - 当测量时间超过最大限制时，输出超时标志

## 11. 使用示例

### 11.1 基本使用

```st
// 测量按钮按下时间
VAR
    Button : BOOL;
    PTM_Instance : PTM;
    PressDuration : TIME;
END_VAR

PTM_Instance(IN := Button);
PressDuration := PTM_Instance.ET;
```

### 11.2 带超时检测的使用

```st
// 测量按钮按下时间，带超时检测
VAR
    Button : BOOL;
    PTM_Instance : PTM;
    PressDuration : TIME;
    Timeout : BOOL;
END_VAR

PTM_Instance(IN := Button);
PressDuration := PTM_Instance.ET;
Timeout := (PressDuration >= T#10S);
```

## 12. 总结

PTM功能块是一个简单但实用的功能块，用于测量输入信号的高电平持续时间。它基于TON定时器实现，通过状态标志管理首次测量和后续测量的行为，能够满足大多数时间测量场景的需求。

通过合理配置和使用，PTM功能块可以为控制系统提供准确的时间测量功能，为工艺控制、设备监控和用户交互等场景提供重要的时间信息。
