# IoT 功能块库说明文档

## 1. 现场设备通信模块

现场设备通信模块是指设备之间通过通信协议进行数据交换的模块，PLC-IOT 支持多种通信方式，如串口通信和 MODBUS 通信，用于连接现场设备和 PLC-IOT 控制系统。

### 1.1 UART

UART 是异步串行通信接口，PLC-IOT 的接口为 5V TTL 电平，通信时需要连接 TXD、RXD 和 GND 三根线。

#### 1.1.1 UART_TX

**功能**：通过 UART 发送数据，支持消息模板和变量替换，可处理字符串和二进制数据。

**参数**：
- **BIN**（输入，BOOL）：指定发送模式，TRUE：二进制，FALSE：默认值：文本
- **MSG**（输入，STRING）：要发送的消息模板，支持使用 [@变量名] 格式嵌入变量
- **T**（输入，BOOL）：触发信号
- **DONE**（输出，BOOL）：完成信号
- **ERR**（输出，BOOL）：错误信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 初始化时设置默认值并创建工作区
- 解析消息模板并构建消息列表，根据 BIN 参数决定使用字符串或二进制
- 当 T 从 FALSE 变为 TRUE 时，将变量转换为消息字符串或二进制数据
- 通过 UART 发送数据
- 发送完成后，DONE 变为 TRUE
- 如有错误，ERR 变为 TRUE

#### 1.1.2 UART_RX

**功能**：通过 UART 接收数据，支持消息解析和变量赋值，可处理字符串和二进制数据。

**参数**：
- **BIN**（输入，BOOL）：指定接收模式，TRUE：二进制，FALSE：默认值：文本
- **MSG**（输入，STRING）：消息模板，用于解析接收到的消息
- **R**（输出，BOOL）：接收完成信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 初始化时设置默认值并创建工作区
- 构建消息列表，根据 BIN 参数决定使用字符串或二进制
- 当接收到 UART 数据时，检查事件
- 解析接收到的消息
- 将解析结果赋值给相应的变量
- 解析完成后，R 变为 TRUE
- 支持处理字符串和二进制数据格式

#### 1.1.3 U0_RX

**功能**：通过 UART0 接收数据，支持消息解析和变量赋值，可处理字符串和二进制数据。

**参数**：
- **BIN**（输入，BOOL）：指定接收模式，TRUE：二进制，FALSE：默认值：文本
- **MSG**（输入，STRING）：消息模板，用于解析接收到的消息
- **R**（输出，BOOL）：接收完成信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 初始化时设置默认值并创建工作区
- 构建消息列表，根据 BIN 参数决定使用字符串或二进制
- 当接收到 UART0 数据时，检查事件
- 解析接收到的消息
- 将解析结果赋值给相应的变量
- 解析完成后，R 变为 TRUE
- 支持处理字符串和二进制数据格式

### 1.2 MODBUS

MODBUS 是一种串行通信协议，用于连接工业设备。

#### 1.2.1 MBUS_MSG（MODBUS 消息）

**功能**：作为 MODBUS 主站，向从站发送命令并接收响应。

**参数**：
- **WR**（输入，BOOL）：写请求信号
- **T**（输入，BOOL）：触发信号
- **SLAVE**（输入，UINT）：从站地址
- **CMD**（输入，UINT）：命令码
- **ADDR**（输入，UINT）：寄存器地址
- **COUNT**（输入，UINT）：寄存器数量
- **DONE**（输出，BOOL）：完成信号
- **ERR**（输出，BOOL）：错误信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 初始化时设置默认值并创建工作区
- 当 T 从 FALSE 变为 TRUE 时，发送 Modbus 消息
- 操作完成后，DONE 变为 TRUE
- 如有错误，ERR 变为 TRUE
- 后台执行 Modbus 通信操作，避免阻塞 PLC 主任务
- 支持 Modbus 主站功能，可发送各种 Modbus 命令

#### 1.2.2 MODBUS 从站

**功能**：作为 MODBUS 从站，接收主站命令并发送响应。

**参数**：
- **ADDR**（输入，BYTE）：从站地址
- **CMD**（输入，BYTE）：命令码
- **REG**（输入/输出，WORD）：寄存器数据
- **COUNT**（输入，WORD）：寄存器数量
- **DONE**（输出，BOOL）：完成信号
- **ERR**（输出，BOOL）：错误信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 接收主站命令
- 根据命令码执行相应操作
- 发送响应数据
- 操作完成后，DONE 变为 TRUE

### 1.3 MODBUS-TCP

MODBUS-TCP 是基于 TCP/IP 协议的 MODBUS 协议变体，用于在以太网网络上传输 MODBUS 消息。

**功能**：通过 TCP/IP 网络进行 MODBUS 通信。

**参数**：
- **IP**（输入，STRING）：IP 地址
- **PORT**（输入，UINT）：端口号
- **WR**（输入，BOOL）：写请求信号
- **T**（输入，BOOL）：触发信号
- **SLAVE**（输入，UINT）：从站地址
- **CMD**（输入，UINT）：命令码
- **ADDR**（输入，UINT）：寄存器地址
- **COUNT**（输入，UINT）：寄存器数量
- **DONE**（输出，BOOL）：完成信号
- **ERR**（输出，BOOL）：错误信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 建立 TCP 连接
- 发送 MODBUS 消息
- 接收响应
- 操作完成后，DONE 变为 TRUE

## 2. 远程通信模块

远程通信模块用于实现设备与远程服务器或其他设备的通信，包括 TCP 和 MQTT 等协议。

### 2.1 TCP

#### 2.1.1 TCP_CLIENT

**功能**：创建 TCP 客户端连接，支持网络状态检测和自动重连。

**参数**：
- **SVR**（输入，STRING）：服务器地址
- **PORT**（输入，UINT）：服务器端口
- **S**（输出，BOOL）：连接状态信号，连接成功时为 TRUE
- **C**（输出，INT）：连接句柄
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 初始化时设置默认值并创建工作区
- 内部使用状态机管理连接过程
- 当网络就绪时，创建 TCP 套接字
- 后台执行连接操作，避免阻塞 PLC 主任务
- 支持域名解析功能，可以使用域名作为服务器地址
- 连接成功后，C 输出连接句柄，S 变为 TRUE，用于给上层调用模块获取是否连接成功的信息
- 支持网络状态检测，当网络断开时会自动关闭连接
- 支持接收数据的缓冲和事件触发机制
- 维护 TCP 客户端连接表，管理多个连接

#### 2.1.2 TCP_TX

**功能**：通过 TCP 发送数据，支持消息模板和变量替换，可处理字符串和二进制数据。

**参数**：
- **BIN**（输入，BOOL）：二进制模式标志
- **C**（输入，INT）：连接句柄
- **MSG**（输入，STRING）：要发送的消息模板，支持使用 [@变量名] 格式嵌入变量
- **T**（输入，BOOL）：触发信号
- **DONE**（输出，BOOL）：完成信号
- **ERR**（输出，BOOL）：错误信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 初始化时设置默认值并创建工作区
- 当 T 从 FALSE 变为 TRUE 时，解析消息模板并构建消息列表
- 根据 BIN 参数决定使用字符串或二进制
- 将变量转换为消息字符串或二进制数据
- 发送数据到指定连接
- 发送完成后，DONE 变为 TRUE
- 如有错误，ERR 变为 TRUE
- 后台执行消息发送操作，避免阻塞 PLC 主任务

#### 2.1.3 TCP_RX

**功能**：通过 TCP 接收数据，支持消息解析和变量赋值，可处理字符串和二进制数据。

**参数**：
- **BIN**（输入，BOOL）：二进制模式标志
- **C**（输入，INT）：连接句柄
- **MSG**（输入，STRING）：消息模板，用于解析接收到的消息
- **R**（输出，BOOL）：接收完成信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 初始化时设置默认值并创建工作区
- 构建消息列表，根据 BIN 参数决定使用字符串或二进制
- 获取 TCP 客户端连接
- 当接收到 TCP 数据时，检查连接状态和事件
- 解析接收到的消息
- 将解析结果赋值给相应的变量
- 解析完成后，R 变为 TRUE
- 支持处理字符串和二进制数据格式

### 2.2 MQTT

#### 2.2.1 WIFI_MQTT

**功能**：管理 WiFi 连接和 MQTT 通信，包括网络状态检测和自动重连。

**参数**：
- **SSID**（输入，STRING）：WiFi SSID
- **PSW**（输入，STRING）：WiFi 密码
- **RST**（输入，BOOL）：重置信号
- **SMART**（输入，BOOL）：智能连接模式
- **SVR**（输入，STRING）：MQTT 服务器地址
- **USR**（输入，STRING）：MQTT 用户名
- **KEY**（输入，STRING）：MQTT 密码
- **LINK**（输出，BOOL）：WiFi 连接状态
- **CONN**（输出，BOOL）：MQTT 连接状态
- **EVT**（输出，BOOL）：事件信号
- **ST**（输出，STRING）：状态信息
- **C**（输出，SINT）：连接计数

**工作原理**：
- 初始化时设置默认值并创建工作区
- 内部使用状态机管理连接过程
- 当网络未就绪时，会自动设置 WiFi 配置
- 当网络就绪后，会自动连接 MQTT 服务器
- LINK 指示 WiFi 连接状态
- CONN 指示 MQTT 连接状态
- 在后台执行耗时操作（如 WiFi 配置和 MQTT 连接），避免阻塞 PLC 主任务
- 支持网络状态检测和自动重连机制

#### 2.2.2 MSG_TX

**功能**：通过 MQTT 发送消息，支持消息模板和变量替换，可处理字符串和二进制数据。

**参数**：
- **TOPIC**（输入，STRING）：MQTT 主题
- **MSG**（输入，STRING）：要发送的消息模板，支持使用 [@变量名] 格式嵌入变量
- **T**（输入，BOOL）：触发信号
- **DONE**（输出，BOOL）：完成信号
- **ERR**（输出，BOOL）：错误信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 初始化时设置默认值并创建工作区
- 当 T 从 FALSE 变为 TRUE 时，解析消息模板并构建消息列表
- 解析消息模板，识别字符串部分和变量部分
- 将变量转换为字符串并替换到消息中
- 发送消息到指定主题
- 发送完成后，DONE 变为 TRUE
- 如有错误，ERR 变为 TRUE
- 在后台执行消息发送操作，避免阻塞 PLC 主任务
- 支持处理字符串格式的消息

#### 2.2.3 MSG_RX

**功能**：通过 MQTT 接收消息，支持消息解析和变量赋值，可处理字符串数据。

**参数**：
- **TOPIC**（输入，STRING）：MQTT 主题
- **MSG**（输入，STRING）：消息模板，用于解析接收到的消息
- **R**（输出，BOOL）：接收完成信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 初始化时设置默认值、创建工作区
- 内部使用状态机管理订阅过程
- 当 MQTT 连接成功后，会自动订阅指定主题
- 当接收到 MQTT 消息时，会处理消息并触发事件
- 当接收到消息时，检查主题是否匹配
- 解析消息模板
- 将消息转换为变量值并赋值给相应的变量
- 解析完成后，R 变为 TRUE
- 支持处理字符串格式的消息

#### 2.2.4 ESPNOW_TX

**功能**：通过 ESP-NOW 发送消息，支持消息模板和变量替换，可处理字符串和二进制数据。

**参数**：
- **BIN**（输入，BOOL）：二进制模式标志，默认为 TRUE
- **DST**（输入，ULINT）：目标设备 MAC 地址
- **MSG**（输入，STRING）：要发送的消息模板，支持使用 [@变量名] 格式嵌入变量
- **T**（输入，BOOL）：触发信号
- **DONE**（输出，BOOL）：完成信号
- **ERR**（输出，BOOL）：错误信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 初始化时设置默认值并创建工作区
- 当 T 从 FALSE 变为 TRUE 时，解析消息模板并构建消息列表
- 根据 BIN 参数决定使用字符串或二进制模式
- 将变量转换为消息字符串或二进制数据
- 检查 ESP-NOW 是否已初始化
- 发送数据到指定目标设备
- 发送完成后，DONE 变为 TRUE
- 如有错误，ERR 变为 TRUE
- 在后台执行消息发送操作，避免阻塞 PLC 主任务

#### 2.2.5 ESPNOW_RX

**功能**：通过 ESP-NOW 接收消息，支持消息解析和变量赋值，可处理字符串和二进制数据。

**参数**：
- **BIN**（输入，BOOL）：二进制模式标志，默认为 TRUE
- **MSG**（输入，STRING）：消息模板，用于解析接收到的消息
- **SRC**（输出，ULINT）：源设备 MAC 地址
- **R**（输出，BOOL）：接收完成信号
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 初始化时设置默认值并创建工作区
- 构建消息列表，根据 BIN 参数决定使用字符串模式或二进制模式
- 当接收到 ESP-NOW 数据时，检查事件
- 从接收到的数据中提取源设备 MAC 地址
- 解析接收到的消息
- 将解析结果赋值给相应的变量
- 解析完成后，R 变为 TRUE
- 支持处理字符串和二进制数据格式

## 3. 系统资源应用模块

系统资源应用模块用于管理系统的各种资源，如实时时钟、非易失性内存、PWM 控制等。

### 3.1 RTC

**功能**：提供实时时钟功能，可读取和设置系统时间。

**参数**：
- **IN**（输入，BOOL）：输入信号，用于设置时间
- **PDT**（输入，DT）：预设日期时间
- **Q**（输出，BOOL）：输出信号
- **CDT**（输出，DT）：当前日期时间

**工作原理**：
- 当 IN 为 TRUE 时，将 PDT 的值设置为系统时间
- CDT 始终输出当前系统时间

### 3.2 NVRAM

**功能**：读写非易失性内存。

**参数**：
- **N**（输入，USINT）：内存地址
- **V**（输入/输出，STRING）：内存值
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 读写指定地址的非易失性内存
- 操作成功后，SUCCESS 变为 TRUE

### 3.3 PWM

**功能**：生成脉冲宽度调制信号。

**参数**：
- **F**（输入，UDINT）：频率
- **P**（输入，USINT）：占空比
- **OUT**（输出，USINT）：PWM 输出
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 根据输入的频率和占空比生成 PWM 信号
- 配置成功后，SUCCESS 变为 TRUE

### 3.4 CLOCK

**功能**：提供时钟功能，可用于定时控制。

**参数**：
- **PT**（输入，TIME）：定时时间
- **Q**（输出，BOOL）：输出信号

**工作原理**：
- 当状态为 0 时，开始计时，Q 变为 TRUE
- 当经过 PT 时间后，Q 变为 FALSE
- 当状态为 2 且再次经过 PT 时间后，状态复位为 0

## 4. 数据处理模块

数据处理模块用于处理各种数据转换和计算任务，支持多种数据类型的转换和处理。

**参数**：

**工作原理**：
- 根据变量名称获取变量的指针和操作函数
- 使用变量的字符串转换函数将变量值转换为字符串
- 将转换后的字符串存储到输出参数 S 中
- 支持多种数据类型，包括 BOOL、SINT、INT、DINT、LINT、USINT、UINT、UDINT、ULINT、REAL、LREAL、TIME、DATE、TOD、DT、STRING、BYTE、WORD、DWORD、LWORD

### 4.2 SETV

- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 将输入的二进制数据转换为十六进制字符串
- 每个字节转换为两个十六进制字符
- 转换成功后，SUCCESS 变为 TRUE

### 4.8 HEX2BIN

**功能**：将十六进制字符串转换为二进制数据。

**参数**：
- **S**（输入，STRING）：十六进制字符串
- **Q**（输出，STRING）：转换后的二进制数据
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 将输入的十六进制字符串转换为二进制数据
- 每两个十六进制字符转换为一个字节
- 转换成功后，SUCCESS 变为 TRUE

### 4.9 CHKSUM8

**功能**：计算8位校验和。

**参数**：
- **S**（输入，STRING）：输入数据
- **Q**（输出，USINT）：校验和值
- **HH**（输出，USINT）：校验和高位（十六进制字符）
- **HL**（输出，USINT）：校验和低位（十六进制字符）
- **SUCCESS**（输出，BOOL）：成功信号

**工作原理**：
- 计算输入数据的8位校验和（所有字节的累加和）
- 将校验和转换为十六进制字符，分别存储到 HH 和 HL 中
- 计算成功后，SUCCESS 变为 TRUE

## 7. 注意事项

1. **使能信号**：所有功能块都包含 EN（使能输入）和 ENO（使能输出）参数，用于功能块的级联控制。

2. **工作区指针**：大多数功能块都包含 `wkr` 指针，用于内部工作区管理，使用时无需手动设置。

3. **通信功能块**：使用通信功能块时，需要确保相关硬件和网络连接正常。

4. **数据转换**：数据转换功能块需要注意输入数据的格式和长度，以确保转换正确。

5. **错误处理**：大多数功能块都提供了错误或成功指示信号，使用时应检查这些信号以确保操作成功。

6. **内存使用**：功能块实例会占用一定的内存空间，在资源受限的系统中应注意合理使用。

7. **UART 通信**：使用 UART 通信时，需要注意波特率、数据位、停止位和校验位的设置，确保与通信设备匹配。

8. **MODBUS 通信**：使用 MODBUS 通信时，需要注意从站地址、寄存器地址和命令码的正确设置。

9. **网络通信**：使用网络通信功能块时，需要确保网络连接正常，并且网络参数设置正确。

10. **系统资源**：使用系统资源功能块时，需要注意资源的合理分配和使用，避免资源冲突。

这些 IoT 功能块为工业自动化系统提供了与物联网设备通信的能力，使得 PLC 系统可以无缝集成到工业物联网生态系统中，实现数据采集、远程监控、智能控制等高级功能。
